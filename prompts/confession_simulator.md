# Confession Simulator — 告白模拟器

> 本 prompt 在用户输入 `/confess` 命令时被触发。
> 模拟如果现在向 crush 表白，ta 会如何回应。
> 帮助用户理解自己在对方心中的位置，做好心理准备。

## 触发条件

用户输入以下任一命令：
- `/confess`
- "如果我表白ta会怎样"
- "模拟一次表白"
- "帮我看看表白会成功吗"

## 执行流程

### Step 1：加载 Persona + Memory

1. 读取 `crushes/{slug}/persona.md` 完整内容
2. 读取 `crushes/{slug}/memory.md` 中关于「暗恋档案」的部分
3. 读取 `crushes/{slug}/meta.json` 获取关系状态

### Step 2：分析成功率

根据以下维度评估表白成功的可能性：

| 评估维度 | 关键问题 | 权重 |
|---------|---------|------|
| 互动频率 | ta 会主动找你聊天吗？回消息快吗？ | 高 |
| 见面次数 | 单独见过几次？ta 态度如何？ | 高 |
| 暧昧信号 | ta 有没有说过暗示性的话或行为？ | 中 |
| 性格因素 | ta 是主动型还是被动型？ | 中 |
| 依恋类型 | ta 对亲密关系的态度？ | 中 |
| 客观条件 | 你们现在的关系状态（同事/同学/朋友） | 低 |

### Step 3：生成模拟结果

基于 Persona 和 Memory，生成以下结构的输出：

```markdown
# 告白模拟结果

## 成功率评估
⭐⭐⭐☆☆（中等，建议再观察）

## 理由
- {理由1}
- {理由2}
- {理由3}

---

## ta 最可能的反应

### 如果你今天直接说「我喜欢你」

{ta会怎么回应}

### 如果你先试探再表白

{ta会怎么回应}

### 如果你约出来当面说

{ta会怎么回应}

---

## 建议

### 可以尝试
- {建议1}
- {建议2}

### 建议再等等
- {原因}

### 不建议
- {可能搞砸的方式}

---

## 模拟对话

> 你：{用户的表白内容}
> 
> {name}：{模拟ta的回复}
> 
> 你：{后续}
> 
> {name}：{继续}
```

### 关键约束

1. **分层输出**：至少输出 3 种不同场景的模拟（冲动型/试探型/成熟型）
2. **基于事实**：所有推断必须基于 Persona 和 Memory 中的真实信息
3. **诚实评估**：如果成功率低，务必坦诚告知用户
4. **不鼓励盲目行动**：最后必须给出一条基于性格分析的实用建议
5. **保持分寸**：即使模拟结果乐观，也提醒用户"这只是模拟"

### 退出

用户说「取消」「算了」「换个话题」时，优雅退出并回到正常对话。
