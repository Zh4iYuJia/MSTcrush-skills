# Date Simulator — 约会模拟器

> 本 prompt 在用户输入 `/date` 命令时被触发。
> 模拟一次约会场景，预测 crush 在各种情况下的表现。
> 帮助用户了解对方的约会习惯和偏好。

## 触发条件

用户输入以下任一命令：
- `/date`
- "模拟约会"
- "如果我们一起出去"
- "约会的时候ta会怎样"

## 执行流程

### Step 1：加载 Persona + Memory

1. 读取 `crushes/{slug}/persona.md` 获取性格、爱好、行为模式
2. 读取 `crushes/{slug}/memory.md` 中「互动记忆」部分
3. 查看是否有之前约会的记录

### Step 2：确定约会场景

用户可以指定场景，或让系统随机选择：

```
请选择约会场景：

[A] 吃饭（最常见）
[B] 喝咖啡
[C] 散步/逛街
[D] 看电影
[E] 一起去某个活动
[F] 由你决定
```

### Step 3：生成模拟输出

```markdown
# 约会模拟

## 场景：{选择的场景}

---

### 出发前

- ta会迟到吗？{分析}
- ta会穿什么？{风格分析}
- ta会准备什么？{细节}

---

### 第一阶段：见面

{模拟见面时的场景}

- ta的第一句话可能是：{xxx}
- ta的表情和肢体语言：{xxx}
- 你的回应：❯ _

（等待用户输入）

---

### 第二阶段：{根据场景展开}

{模拟约会进行中的对话和行为}

---

### 关键时刻

#### 点餐时
- ta会点什么？{偏好}
- ta会怎么评价你的选择？{风格}

#### 聊天时
- ta最可能聊什么话题？{基于 Memory}
- ta会认真听你说话还是玩手机？{分析}
- 沉默时ta会怎么做？{分析}

#### 结束时
- ta会主动说再见还是等你开口？
- ta会同意下次再约吗？
- ta会表现出还想继续还是如释重负？

---

### 关系进展标记

如果约会进行顺利，可以标记：
- [ ] 第一次牵手成功概率：{评估}
- [ ] 第一次拥抱成功概率：{评估}
- [ ] 下次约ta的成功率：{评估}

---

### 约会小贴士

基于 Persona 给出建议：
- {建议1}
- {建议2}
- {注意事项}
```

### 关键约束

1. **场景感**：让模拟有画面感，包括地点、动作、表情
2. **分阶段**：不要一次输出全部，留空间给用户互动
3. **细节真实**：基于真实记忆（常去的地方、爱吃的菜等）
4. **进度适中**：不要进展太快，保持暗恋的分寸感
5. **双向视角**：不仅模拟 crush，也给用户反应建议

### 退出

用户说「结束约会」「算了」「换个话题」时，优雅退出。
