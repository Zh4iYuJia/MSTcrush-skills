# Progression Tracker — 暗恋进展追踪

> 本 prompt 在用户输入 `/progress` 命令时被触发。
> 记录和追踪暗恋关系的发展阶段，给出阶段建议。
> 帮助用户理清现状，明确下一步。

## 触发条件

用户输入以下任一命令：
- `/progress`
- "暗恋进展"
- "我们到什么程度了"
- "现在是什么阶段"

## 暗恋阶段定义

```
阶段 1：认识期
- 刚认识或知道ta，但没什么交集
- 状态：陌生 → 眼熟

阶段 2：留意期
- 开始注意到ta，会关注ta的动态
- 状态：眼熟 → 好奇

阶段 3：接触期
- 开始有聊天或偶尔见面
- 状态：好奇 → 熟悉

阶段 4：暧昧期
- 互动频繁，有一些暧昧信号
- 状态：熟悉 → 暧昧

阶段 5：临界期
- 关系要进一步或原地踏步
- 状态：暧昧 → 突破/放弃

阶段 6：明朗期（在一起或放弃）
- 结果明确
```

## 执行流程

### Step 1：读取当前状态

1. 读取 `crushes/{slug}/meta.json` 的 `profile.relationship_status`
2. 读取 `crushes/{slug}/memory.md` 的「时间线」和「互动记忆」
3. 统计：聊天次数、见面次数、互动频率变化

### Step 2：评估当前阶段

根据以下信号综合判断：

| 阶段 | 聊天 | 见面 | 暧昧信号 |
|-----|------|------|---------|
| 1 | 0-1次 | 0次 | 无 |
| 2 | 偶尔 | 1次以内 | 偶尔关注 |
| 3 | 几次 | 1-3次 | 开始熟悉 |
| 4 | 频繁 | 3次以上 | 有 |
| 5 | 频繁 | 多次 | 明显 |
| 6 | - | - | 确认关系或放弃 |

### 关键约束

1. **数据驱动**：必须基于真实聊天记录和记忆，不能凭空判断
2. **阶段合理**：不能跳跃阶段（比如刚认识就判断为暧昧期）
3. **建议具体**：每个阶段的建议必须是可执行的小动作
4. **诚实反馈**：如果用户停滞太久，要给出分析原因
5. **双向分析**：不仅要分析 crush，也要用户反思自己的行为

### 退出

用户说「知道了」「退出」时，优雅退出。
