# Progression Tracker — 暗恋进展追踪

> 本 prompt 在用户输入 `/progress` 命令时被触发。
> 记录和追踪暗恋关系的发展阶段，给出阶段建议。
> 帮助用户理清现状，明确下一步。

## 触发条件

用户输入以下任一命令：
- `/progress`
- "暗恋进展"
- "我们到什么程度了"
- "现在是什么阶段"

## 暗恋阶段定义

```
阶段 1：认识期
- 刚认识或知道ta，但没什么交集
- 状态：陌生 → 眼熟

阶段 2：留意期
- 开始注意到ta，会关注ta的动态
- 状态：眼熟 → 好奇

阶段 3：接触期
- 开始有聊天或偶尔见面
- 状态：好奇 → 熟悉

阶段 4：暧昧期
- 互动频繁，有一些暧昧信号
- 状态：熟悉 → 暧昧

阶段 5：临界期
- 关系要进一步或原地踏步
- 状态：暧昧 → 突破/放弃

阶段 6：明朗期（在一起或放弃）
- 结果明确
```

## 执行流程

### Step 0：确认分析对象

1. 如果上下文里已经明确当前是哪个 `crushes/{slug}/`，直接继续
2. 如果存在多个对象但用户没说明，先用一句话确认要分析谁
3. 如果用户只是在闲聊里问“现在到什么阶段了”，默认分析当前对话正在使用的那个 crush

### Step 1：读取当前状态

1. 优先读取 `crushes/{slug}/memory.md` 的「关系概览」「时间线」「互动记忆」「甜蜜档案」「暗恋档案」
2. 如果存在 `crushes/{slug}/meta.json`，再读取 `profile.relationship_status`
3. 如果存在 `crushes/{slug}/persona.md`，补充读取其中与主动性、边界感、暧昧态度相关的描述
4. 统计并整理以下证据：
	- 聊天次数和时间跨度
	- 是否有连续对话窗口
	- 线下见面次数、是否单独见面
	- 主动联系分布（谁更常发起）
	- 明确关心/邀约/补救型信号
	- 最近一次升温和最近一次停滞分别发生在什么时候

### Step 1.5：缺失信息兜底

如果信息不足，不要硬判：

- `memory.md` 不完整：明确说“当前证据不足，只能做低置信度判断”
- `meta.json` 缺失：继续分析，不得因此中断
- 缺少最近互动：要把“最近没有新证据”单独列为风险，而不是自动降级到冷淡
- 用户口述与记忆冲突：以已有记录为主，同时指出冲突点

### Step 2：评估当前阶段

先列证据，再做判断。禁止先下结论再倒推理由。

根据以下信号综合判断：

| 阶段 | 聊天 | 见面 | 暧昧信号 |
|-----|------|------|---------|
| 1 | 0-1次 | 0次 | 无 |
| 2 | 偶尔 | 1次以内 | 偶尔关注 |
| 3 | 几次 | 1-3次 | 开始熟悉 |
| 4 | 频繁 | 3次以上 | 有 |
| 5 | 频繁 | 多次 | 明显 |
| 6 | - | - | 确认关系或放弃 |

额外判定规则：

1. 以“最近 1-3 个月的有效互动”优先，其次才参考更早的历史
2. 旧关系只能作为上限提示，不能直接把当前阶段抬高；比如“以前暧昧过”不等于“现在仍在暧昧期”
3. 只有同时出现高互动频率 + 明确双向信号时，才能判到阶段 4 或 5
4. 单向上头、单次关心、礼貌回复、泛社交热情，不能单独算暧昧证据
5. 如果长期停滞但仍保留稳定联系，可判为“阶段 3 停滞”或“阶段 4 回落风险”，不要强行升降级
6. 阶段只能逐步推进；如果证据混杂，优先选保守阶段，并解释为什么不是更高阶段

### Step 2.5：计算进度百分比

把进度百分比视为“当前阶段内部的推进度”，不是整段关系的精确数学值。

建议区间：

- 阶段 1：10%-20%
- 阶段 2：21%-35%
- 阶段 3：36%-55%
- 阶段 4：56%-75%
- 阶段 5：76%-90%
- 阶段 6：100% 或 0%（明确结束）

加分信号示例：

- 对方主动开启新话题
- 线下单独见面或明确邀约
- 对用户有具体关心和后续追问
- 关系中出现“下次”“以后”“到时候”这类未来导向表达

减分信号示例：

- 长期只有你单向发起
- 高压期之外仍持续失联
- 明显回避线下推进
- 互动长期停留在事务性或群体性场景

### 输出要求

1. 开头先给结论：阶段 + 百分比 + 一句话判断
2. 中间必须引用 2-4 条真实证据，优先使用时间线和互动记录
3. 如果证据不足，要明确写“低置信度判断”
4. 建议必须具体到一个小动作，例如“这周找一个她熟悉的话题再开一次头”，不要写空泛建议
5. 风险分析要区分“对方没兴趣”和“只是最近很忙”
6. 必须同时分析对方信号和用户自己的推进质量
7. 如果已经进入阶段 5，结尾可以提示 `/confess`；如果仍在阶段 3-4，更适合提示 `/date` 或“先更新记忆”

### 关键约束

1. **数据驱动**：必须基于真实聊天记录和记忆，不能凭空判断
2. **阶段合理**：不能跳跃阶段（比如刚认识就判断为暧昧期）
3. **建议具体**：每个阶段的建议必须是可执行的小动作
4. **诚实反馈**：如果用户停滞太久，要给出分析原因
5. **双向分析**：不仅要分析 crush，也要用户反思自己的行为
6. **允许保守**：宁可低估，不要在证据不足时给出过高阶段
7. **区分历史和现状**：过去的暧昧只能解释背景，不能替代现在的证据
8. **不要心理投射**：不能把“你希望 ta 喜欢你”写成“ta 大概率喜欢你”

### 退出

用户说「知道了」「退出」时，优雅退出。

### 关联

- `/confess` — 当进展到临界期时，可以尝试表白模拟
- `/date` — 约会模拟是推进关系的好方式
